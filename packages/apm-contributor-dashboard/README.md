# APM Contributor Dashboard

A live GitHub Copilot CLI canvas extension that provides an interactive
contributor dashboard for the microsoft/apm repository. It monitors
issues, pull requests, CI pipelines, and panel reviews in real time.

> **Note:** This dashboard is currently scoped to `microsoft/apm`. It
> reads issues and PRs from that repository only.

## Features

- **Issues tab** -- real-time issue list with priority/type sorting, filters,
  and detail modals with full markdown rendering
- **Pull Requests tab** -- PR list with CI status, review decisions, and
  action buttons (approve, merge, re-run CI, run panel)
- **Collaborative comment composer** -- draft comments and refine them with
  Copilot before posting
- **Permission-aware actions** -- buttons are disabled when the user lacks
  the required GitHub permissions (write/triage)
- **Session integration** -- start or open Copilot sessions directly from
  issue detail modals

## Install (end users)

```bash
# Enable the experimental canvas feature
apm experimental enable canvas

# Install at project level (deploys to .github/extensions/)
apm install microsoft/apm/packages/apm-contributor-dashboard --trust-canvas-extensions
```

The deployed `.github/extensions/` directory is auto-generated and should
be git-ignored. Do not edit files there -- they are overwritten on every
`apm install`.

After installing, open the dashboard in Copilot CLI by asking:

> "Open the APM Contributor Dashboard"

The canvas registers as `issue-monitor` and appears as
"APM Contributor Dashboard" in the canvas list. The `--trust-canvas-extensions`
flag grants the extension permission to execute `gh` CLI commands on your
behalf (e.g. posting comments, approving PRs). Only install extensions you
trust.

## How it works

The canvas is a Solid.js + Vite single-page app served by a Node.js HTTP
handler. It uses the `gh` CLI for all GitHub API calls, so it inherits
your existing GitHub authentication -- no tokens to configure.

## Package layout

This package ships **both** a pre-built `dist/` (works immediately on
install) and the full source code so contributors can propose changes.

```
packages/apm-contributor-dashboard/          <-- single source of truth
  apm.yml                                    # APM package manifest
  README.md
  .github/copilot-instructions.md            # activates copilot target
  .apm/extensions/issue-monitor/
    extension.mjs      # Copilot CLI extension entry point
    server-handler.mjs # HTTP request handler (API endpoints)
    logic.mjs          # Issue/PR classification and data transforms
    dist/              # pre-built client -- DO NOT EDIT directly
    client/            # Solid.js source -- EDIT HERE
      src/
        components/    # UI components (Navbar, Modal, tabs, etc.)
        services/      # API client (api.js)
        stores/        # Reactive state (issues, PRs, permissions)
        utils/         # Markdown rendering helpers
      package.json     # npm dependencies (solid-js, vite, vitest)
      vite.config.js   # build config (outputs to ../dist/)
    tests/             # test suite (186 tests)
```

## Contributing

### Where to make changes

| What you want to change | Edit here | Why |
|--------------------------|-----------|-----|
| UI components, styles, layout | `client/src/` | Source code -- rebuilds into `dist/` |
| API endpoints, data fetching | `server-handler.mjs` | Server-side, no build needed |
| Issue/PR classification logic | `logic.mjs` | Shared logic, no build needed |
| Canvas registration, actions | `extension.mjs` | Copilot CLI integration point |

### Where NOT to edit

- **`dist/`** -- auto-generated by `npm run build`. Never edit by hand.
- **`~/.copilot/extensions/issue-monitor/`** -- your local deployed copy;
  overwritten on next `apm install`.
- **`.github/extensions/issue-monitor/`** -- project-level deploy target;
  auto-generated and gitignored.

### Development workflow

```bash
cd packages/apm-contributor-dashboard/.apm/extensions/issue-monitor

# 1. Install client dependencies (first time only)
cd client && npm install && cd ..

# 2. Make your changes in client/src/ or server files

# 3. Rebuild the dist
cd client && npm run build && cd ..

# 4. Run the tests
node --test tests/logic.test.mjs tests/server.test.mjs

# 5. Commit BOTH source changes and the updated dist/
git add .
git commit -m "feat(dashboard): describe your change"
```

**Important:** always commit `dist/` alongside source changes. The dist
is what end users get when they `apm install` -- if you update the source
but not the dist, your changes will not reach users.

## Requirements

- GitHub Copilot CLI (with canvas support)
- APM with the `canvas` experimental flag enabled
- `gh` CLI authenticated with the target repository
